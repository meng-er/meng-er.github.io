  • 版本说明

  • Q1：取不到前端请求的参数->A：body-parser的使用

首次在后端console.log(req)，找不到前端发送过来的参数。

查看express文档后发现取到post方法的参数需要使用中间件。

express内置了express.json([options])（仅支持v4.16.0 以上），用来解析json格式的请求。同时该中间件是基于node提供的body-parser中间件，所以最后选择使用body-parser。

安装

引入

这个模块一共提供了四个解析器

  • JSON body parser
  • Raw body parser
  • Text body parser
  • URL-encoded form body parser

猜测这里只需要用到json的解析器

哦豁报错了，说req太大了

PayloadTooLargeError: request entity too large

于是修改了一下前端，post了几个字符串过来，哎你猜怎么着，后端取到啦

看来取参应该是没问题了，传不过来是文件太大的问题

  • Q2:PayloadTooLargeError: request entity too large->A：提高limit

关于传输大小的问题，body-parser的文档里有提到，如果有需要在body-parser.json({option})里修改选项

原文：

limit

Controls the maximum request body size. If this is a number, then the value specifies the number of bytes; if it is a string, the value is passed to the bytes library for parsing. Defaults to '100kb'.

  • 新建数据库

    create database 数据库名;

    如新建名字为node_project的数据库

    create database node_project;

  • 新建表

    use database_name
